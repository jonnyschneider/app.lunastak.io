## Baseline-v1 (2025-12-13)

**Type:** Feature Release - Experimentation Foundation
**Status:** Production

### Overview

Establishes **baseline-v1** as our normalized control variant for experimentation. After extensive testing revealed "wooden" strategy outputs, we've simplified the user experience, removed prescriptive elements, and implemented comprehensive measurement infrastructure to systematically improve strategy quality through experimentation.

### What's New

**Simplified User Experience**

- **Streamlined Conversation Flow** - Removed lens selection step; conversations go directly to strategic questions
- **Multi-line Input** - 8-row textarea replaces single-line field; Cmd/Ctrl+Enter to send
- **Full-Height Chat** - Chat now uses full viewport height on all devices
- **Three-Option Extraction** - Continue now / Flag for later / Dismiss at extraction confirmation

**Honest About Capabilities**

- Vision, Mission, Objectives: Generated by AI ✅
- Initiatives, Principles: Honest empty states with CTAs (not generated yet)
- Measures demand for features before building them

**In-App Learning**

- Info icons (ℹ️) on every Decision Stack element
- Explains what makes "Good" vs "Wooden" statements
- Strategic thinking guidance in collapsible accordion

**Simplified Feedback**

- Single mechanism: "Was this helpful?" (Helpful / Not Helpful)
- Quality assessment separated for internal research use

### Under the Hood

**Experimentation Infrastructure**

- All conversations tagged with `experimentVariant: 'baseline-v1'`
- Event logging tracks user interactions (fake doors, info views, choices)
- Quality ratings (good/bad) for internal evaluation
- Enables systematic A/B testing against future variants

**Analysis Tools**

- 6 new Jupyter notebook cells for event analysis
- Enhanced TraceAnalyzer with event querying
- Feature demand measurement
- Quality correlation analysis
- Variant comparison capabilities

### What's Coming Next

**Planned Experiments:**
1. **Emergent Theme Extraction** - Let themes emerge naturally vs prescriptive fields
2. **Energetic Generation Prompts** - Add authenticity guidance to reduce wooden outputs
3. **Lens Inference** - Infer strategic focus from conversation content

### Technical Details

- New Event table for interaction tracking
- Quality rating fields on Trace table
- Experiment variant field on Conversation table
- No breaking changes - backward compatible

### Migration

```bash
npx prisma db push         # Update database schema
npx prisma generate        # Regenerate Prisma client
rm -rf .next              # Clear Next.js cache (if needed)
```

## R1 - Learning-First Foundation (2025-12-09)

**Status:** Private Beta

### What is R1?

Decision Stack R1 is a **learning experiment** in AI-powered strategic planning. You'll have a conversation about your business, and the AI will generate your Vision, Mission, and Objectives.

But more importantly: we're building a **systematic quality improvement system** based on LLM-as-Judge methodology. R1 is our data collection phase.

### What You'll Experience

**Simple Conversation Flow:**
1. AI asks 3-10 strategic questions about your business
2. You choose a strategic lens (Competitive Advantage, Customer-Centric, Innovation-Driven, etc.)
3. AI explores that dimension deeply
4. Review extracted insights, choose to generate or keep exploring
5. Get Vision, Mission, and 3 SMART Objectives

**Deliberately Minimal UI:**
We're focusing on conversation quality, not features. No saving, no history, no editing. Just the core loop.

### What We're Learning

**From You:**
- Which questions and communication styles work best?
- Do different strategic lenses lead to similar conclusions?
- Are outputs grounded in the conversation or generic?

**Through LLM-as-Judge:**
We're building AI that evaluates AI quality. Here's how:
1. Evaluate conversations with full context
2. Manually review failures to discover patterns
3. Build LLM judges to automatically detect those patterns
4. Measure improvement quantitatively
5. Iterate based on data, not gut feel

**Your role:** Rate every output (helpful/not helpful). Your feedback is our primary quality signal.

### What's Missing (Intentionally)

- No user accounts (temporary session IDs only)
- No conversation history
- No editing or refining outputs
- No exporting or sharing

**Why?** We're validating core assumptions first: Can AI have useful strategic conversations? Can we measure and improve quality systematically?

### What's Next

**Phase 2:** Manual error analysis of all R1 traces. Build first LLM judges for top failure modes.

**Phase 3:** Feature expansion based only on what Phase 2 data validates.

**Long-term:** An AI strategic partner that continuously improves through rigorous quality measurement. We're building the measurement system first, features second.

### Thank You

You're not just testing software - you're participating in an experiment in systematic AI quality improvement.

Every conversation, every rating, every piece of feedback makes the next version better.

Thanks for learning with us!
